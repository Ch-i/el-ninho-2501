The shape of it

Five links, each a small misplaced trust — in old commits, in file extensions, in a shared password, and finally in a filename:

vhost fuzz → git.nexus.htb (Gitea) · billing.nexus.htb (Krayin CRM) └─ Gitea repo admin/krayin-docker-setup → git history leaks a DB password └─ password reuse → Krayin CRM admin (j.matthew) └─ TinyMCE /admin/tinymce/upload keeps .php → webshell → www-data └─ live .env DB pass → reused on SSH → jones → user.txt └─ root timer gitea-template-sync.py: os.path.join(stage, tree_path) └─ git tree path ../../../../../root/.ssh/authorized_keys → root
01 · recon

Two ports, three vhosts

Only 22 and 80 are open; nginx 302-redirects to nexus.htb, so it's virtual-hosted. Fuzzing the Host header surfaces two more names — a Gitea instance and a Krayin CRM.

02 · loot
02

Blanked, but not gone

The docker-setup repo's .env looks scrubbed — but the previous commit still holds a real DB_PASSWORD. Git never forgets.

Takeaway

Never trust a redaction commit. git log -p --all and tools like trufflehog/gitleaks read the whole history, not just HEAD.

03 · web RCE
03

An editor that keeps your .php

The leaked password logs j.matthew@nexus.htb into Krayin CRM. Its TinyMCE image uploader at /admin/tinymce/upload validates nothing — hand it a .php and it stores it under /storage/tinymce/ and returns the URL. That URL is a webshell.

04 · foothold
04

The real password lives on disk

As www-data, the live /var/www/krayin/.env holds a different DB password than the git one. /etc/passwd shows a real user jones — and the password is reused.

05 · root
05

A root cron that trusts filenames

A systemd timer runs /etc/gitea/template-sync.py as root every minute. For each Gitea repo marked template, it reads the git tree and writes every blob to a staging dir with:

# template-sync.py (runs as root)
target = os.path.join(stage_path, filepath)   # filepath comes straight from `git ls-tree`
os.makedirs(os.path.dirname(target), exist_ok=True)
with open(target, 'wb') as f: f.write(blob)

Git blob paths aren't sanitized, so a tree entry named ../../../../../root/.ssh/authorized_keys escapes staging. Normal git add refuses .., but low-level git mktree builds the tree by hand. Push it, flag the repo a template, wait one minute.

Closing Nexus

  • Purge secrets from git history, don't just blank them in a new commit; rotate anything committed.
  • Validate uploads by re-encoding images and storing them outside the web root / with a non-exec extension.
  • Never reuse a DB password for a shell account.
  • Sanitize archive/tree extraction (the "zip-slip" class): resolve paths and confirm they stay inside the target dir.
